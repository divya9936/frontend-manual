⚙️ How to Contribute

The GitHub Workflow

1

Fork the Repository

Click "Fork" on the GitHub repo to create your own copy.

2

Add Your Lesson to data.js

Edit data.js and add your lesson following the existing format. See the CONTRIBUTING.md file for the exact structure.

3

Commit and Push

Commit your changes with a clear message: "Add lesson: [Your Lesson Title]"

4

Submit a Pull Request

Open a PR from your fork to the main repo. I'll review it and merge if it meets our guidelines.

5

Your Lesson Goes Live

Once merged, your lesson appears on Frontend Manual and you're credited as a contributor.

📝 Lesson Format

What We're Looking For

Each lesson should be concise, practical, and include a code example. Add your lesson to the wisdomData array in data.js using this format: 

{
  id: 99,
  category: "react",
  icon: "⚛️",
  difficulty: "intermediate",
  title: "Your lesson title",
  body: "Clear explanation (50-100 words) of what to do and why",
  code: `// Bad practice
const bad = () => {...}

// Good practice  
const good = () => {...}`,
  tags: ["react", "general"]
}

Before You Submit

  • ✅ Be specific — real lessons from real projects, not generic advice
  • ✅ Include code — show the problem and the solution
  • ✅ Be practical — focus on actionable, hands-on knowledge
  • ✅ Be honest — share real experiences from your work
  • ✅ Keep it concise — most lessons fit in 50-100 words plus code
  • ❌ Avoid self-promotion — this isn't a marketing platform
  • ❌ Avoid duplicates — check existing lessons first
  • ❌ Avoid outdated tech — focus on current best practices
